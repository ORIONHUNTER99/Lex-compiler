# Lex Error Codes Reference

This document describes all error codes used by the Lex compiler.

## Error Code Format

Error codes follow the format: `[CATEGORY][NUMBER]`

- **Category**: Single letter indicating the error category
- **Number**: Three-digit sequential number

## Categories

| Prefix | Category | Severity |
|--------|----------|----------|
| **F** | File | Error |
| **L** | Lexer | Error |
| **P** | Parser | Error |
| **M** | Module | Error/Info |
| **W** | Warning | Warning |
| **E** | Validation | Error |

---

## File Errors (F)

### F001 - Cannot Open File

**Severity**: Error

**Message**: `Cannot open file: {filepath}`

**Cause**: The specified file does not exist or is not readable.

**Solution**: Check that the file path is correct and the file has read permissions.

**Example**:
```
Error [F001]: Cannot open file: missing.lex
  Suggestion: Check that the file exists and is readable
```

---

## Lexer Errors (L)

### L001 - Lexer Error

**Severity**: Error

**Message**: Various (passed from lexer)

**Cause**: Invalid token, unexpected character, or malformed input.

**Common causes**:
- Unterminated string literal
- Invalid character in identifier
- Malformed number literal

**Example**:
```
Error [L001]: Unexpected character '@' at line 5, column 10
```

---

## Parser Errors (P)

### P001 - Parser Error

**Severity**: Error

**Message**: Various (passed from parser)

**Cause**: Syntax error in the Lex source file.

**Common causes**:
- Missing closing brace `}`
- Missing colon after property name
- Invalid definition syntax
- Unexpected token

**Example**:
```
Error [P001]: Expected '}' but found 'structure' at line 12
```

---

## Module Errors (M)

### M001 - Cannot Find Imported File

**Severity**: Error

**Message**: `Cannot find imported file: {path}`

**Cause**: An `import` statement references a file that does not exist.

**Solution**: Check the import path is correct relative to the importing file.

**Example**:
```
Error [M001]: Cannot find imported file: ./units/game_units.lex
  Suggestion: Check the import path is correct
```

### M002 - Skipped Internal Definition

**Severity**: Info (verbose mode only)

**Message**: `Skipped internal definition: {identifier}`

**Cause**: A definition marked as `internal` or `private` was skipped during compilation in modder mode.

**Solution**: Use `--allow-internal` flag to include internal definitions.

**Example**:
```
Info [M002]: Skipped internal definition: _debug_helper
```

---

## Warnings (W)

### W001 - Unknown Target

**Severity**: Warning

**Message**: `Unknown target: {target_name}`

**Cause**: The specified output target is not recognized.

**Valid targets**: `lua`, `json`, `gd` (Godot), `cs` (Unity), `love2d`, `defold`, `ts` (TypeScript)

**Example**:
```
Warning [W001]: Unknown target: python
```

### W002 - Path Resolution Warning

**Severity**: Warning

**Message**: `Could not resolve canonical path for '{path}'`

**Cause**: The filesystem cannot resolve the canonical path, possibly due to symlinks or permission issues.

**Solution**: The compiler will use the original path. Usually not critical.

---

## Validation Errors (E)

Validation errors are generated by the semantic validator and use codes specific to each validation rule. These codes are defined in the schema and validator components.

## Error Handling Best Practices

1. **Always check `result.success`** before using compiler output
2. **Process all errors**, not just the first one
3. **Use verbose mode** (`options.verbose = true`) for additional context
4. **Handle warnings** separately from errors - compilation may succeed with warnings

```cpp
auto result = lex::compile(source, options);

if (!result.success) {
    for (const auto& err : result.errors) {
        std::cerr << "Error [" << err.code << "]: " << err.message;
        if (!err.location.empty()) {
            std::cerr << " at " << err.location;
        }
        std::cerr << std::endl;
        if (!err.suggestion.empty()) {
            std::cerr << "  Suggestion: " << err.suggestion << std::endl;
        }
    }
}

for (const auto& warn : result.warnings) {
    std::cerr << "Warning [" << warn.code << "]: " << warn.message << std::endl;
}
```
