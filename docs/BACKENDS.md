# Lex Backends

Complete list of supported code generation targets.

## Free Backends (Lex Public)

These backends are included in Lex — free forever under MIT license.

| Backend | Output | Target Engine |
|---------|--------|---------------|
| **lua** | `.lua` | General purpose, game frameworks |
| **json** | `.json` | Data files, any engine |
| **godot** | `.gd` | Godot Engine 4.x |
| **unity** | `.cs` | Unity game development |
| **love2d** | `.lua` | LÖVE2D framework |
| **defold** | `.lua` | Defold engine |

### Usage

```bash
# Single target
lexc game.lex -t lua

# Multiple targets
lexc game.lex -t lua,json,godot

# With output directory
lexc game.lex -t unity -o ./output/
```

## Backend Comparison

### Script Backends (Lua, GDScript, Python)

```lua
-- Generated by Lex (lua backend)
local GameData = {}

GameData.characters = {
    warrior = {
        name = "Warrior",
        hp = 100,
        attack = 15
    }
}

return GameData
```

### Class Backends (C#, C++, Rust)

```csharp
// Generated by Lex (unity backend)
public static class GameData
{
    public class Character
    {
        public string name;
        public int hp;
        public int attack;
    }

    public static readonly Character Warrior = new Character
    {
        name = "Warrior",
        hp = 100,
        attack = 15
    };
}
```

### Data Backends (JSON)

```json
{
    "characters": {
        "warrior": {
            "name": "Warrior",
            "hp": 100,
            "attack": 15
        }
    }
}
```

## Custom Backends

You can create custom backends by implementing the `Backend` interface:

```cpp
class MyBackend : public lex::Backend {
public:
    std::string name() const override { return "mybackend"; }
    std::string file_extension() const override { return ".ext"; }
    std::string generate(const AST& ast) override {
        // Your code generation logic
    }
};

// Register in BackendRegistry
BackendRegistry::instance().register_backend("mybackend", 
    []() { return std::make_unique<MyBackend>(); });
```
