# Unity Integration Guide

This guide shows how to use Lex with Unity to generate C# ScriptableObjects for your game content.

## Quick Start

### 1. Install Lex

```bash
# Windows
irm https://raw.githubusercontent.com/David-Imperium/Lex-compiler/main/install.ps1 | iex

# Linux/macOS
curl -fsSL https://raw.githubusercontent.com/David-Imperium/Lex-compiler/main/install.sh | bash
```

### 2. Create Your Content

Create a file `content/items.lex`:

```lex
item Sword {
    name: "Iron Sword"
    rarity: "common"
    damage: 10
    cost: { Gold: 50 }
}

item Shield {
    name: "Wooden Shield"
    rarity: "common"
    defense: 5
    cost: { Gold: 30 }
}

item LegendaryBlade {
    name: "Excalibur"
    rarity: "legendary"
    damage: 50
    cost: { Gold: 1000, Gem: 10 }
}
```

### 3. Compile to C#

```bash
lexc content/items.lex --types item -t cs -o Assets/Scripts/Generated/
```

This generates `Assets/Scripts/Generated/items.cs`:

```csharp
// Auto-generated by Lex Compiler
using UnityEngine;
using System.Collections.Generic;

[CreateAssetMenu(fileName = "ItemData", menuName = "Game/Item")]
public class ItemData : ScriptableObject {
    public string id;
    public string name;
    public string rarity;
    public int damage;
    public int defense;
    public ResourceCost cost;
}

[System.Serializable]
public class ResourceCost {
    public int Gold;
    public int Gem;
}
```

### 4. Create ScriptableObject Assets

In Unity:
1. Right-click in Project window
2. Create → Game → Item
3. Name it "Sword" and fill in the values

Or create a loader that instantiates from the generated code:

```csharp
// Assets/Scripts/ContentLoader.cs
using UnityEngine;
using System.Collections.Generic;

public static class ContentLoader
{
    private static Dictionary<string, ItemData> items;
    
    public static void LoadItems()
    {
        items = new Dictionary<string, ItemData>();
        
        // Load all ItemData assets from Resources
        var loadedItems = Resources.LoadAll<ItemData>("Items");
        foreach (var item in loadedItems)
        {
            items[item.id] = item;
        }
    }
    
    public static ItemData GetItem(string id)
    {
        return items.TryGetValue(id, out var item) ? item : null;
    }
}
```

### 5. Use in Your Game

```csharp
// PlayerInventory.cs
using UnityEngine;

public class PlayerInventory : MonoBehaviour
{
    public void AddItem(string itemId)
    {
        var itemData = ContentLoader.GetItem(itemId);
        if (itemData != null)
        {
            Debug.Log($"Added {itemData.name} to inventory");
            // Add to inventory logic
        }
    }
}
```

## Unity-Specific Features

### Resource Costs

Lex generates a `ResourceCost` helper class:

```csharp
[System.Serializable]
public class ResourceCost {
    public int Gold;
    public int Wood;
    public int Coal;
    public int Steel;
    public int Energy;
    public int Research;
    public int Industry;
}
```

### Conditions

Conditions are stored as strings. Implement an evaluator:

```csharp
// GameState.cs
using UnityEngine;
using System.Collections.Generic;

public class GameState : MonoBehaviour
{
    public HashSet<string> technologies = new HashSet<string>();
    public HashSet<string> structures = new HashSet<string>();
    public Dictionary<string, int> resources = new Dictionary<string, int>();
    
    public bool HasTechnology(string techId) => technologies.Contains(techId);
    public bool HasStructure(string structId) => structures.Contains(structId);
}

// LexHelpers.cs (auto-generated)
public static class LexHelpers
{
    public static bool HasTechnology(string techId, GameState state)
    {
        return state != null && state.technologies.Contains(techId);
    }
    
    public static bool EvaluateCondition(string expr, GameState state)
    {
        if (string.IsNullOrEmpty(expr)) return true;
        
        expr = expr.Trim();
        if (expr.StartsWith("has_technology(") && expr.EndsWith(")"))
        {
            string id = expr.Substring(14, expr.Length - 15);
            return HasTechnology(id, state);
        }
        return true;
    }
}
```

### Multiple Types

Compile multiple definition types:

```lex
character Hero {
    name: "Arthur"
    class: "Warrior"
    level: 1
}

item Sword {
    name: "Excalibur"
    damage: 50
}

quest SaveVillage {
    name: "Save the Village"
    reward: { Gold: 500, XP: 100 }
}
```

```bash
lexc content/game.lex --types character,item,quest -t cs -o Assets/Scripts/Generated/
```

Generates three classes:
- `CharacterData`
- `ItemData`
- `QuestData`

## Best Practices

### 1. Generated Folder Structure

```
Assets/
├── Scripts/
│   ├── Generated/        # Lex-generated code
│   │   ├── Items.cs
│   │   ├── Characters.cs
│   │   └── Quests.cs
│   ├── ContentLoader.cs
│   └── GameState.cs
├── Resources/
│   ├── Items/           # ScriptableObject assets
│   ├── Characters/
│   └── Quests/
└── Scenes/
```

### 2. Don't Edit Generated Files

Generated files are overwritten on each compile. Create separate partial classes if needed:

```csharp
// ItemDataExt.cs (not generated)
public partial class ItemData
{
    public bool IsLegendary => rarity == "legendary";
    public int TotalStats => damage + defense;
}
```

### 3. Runtime vs Design-Time

**Design-Time**: Create ScriptableObject assets in Unity Editor
**Runtime**: Parse JSON output if you need to load content dynamically:

```bash
lexc content.lex -t json -o StreamingAssets/
```

```csharp
// Load JSON at runtime
var json = File.ReadAllText(Path.Combine(Application.streamingAssetsPath, "content.json"));
var data = JsonUtility.FromJson<GameData>(json);
```

## Build Pipeline Integration

Add Lex compilation to your build process:

```csharp
// Editor/ContentBuildPipeline.cs
using UnityEditor;
using UnityEditor.Build;
using UnityEditor.Build.Reporting;
using System.Diagnostics;

class ContentBuildPipeline : IPreprocessBuildWithReport
{
    public int callbackOrder => 0;
    
    public void OnPreprocessBuild(BuildReport report)
    {
        var process = new Process
        {
            StartInfo = new ProcessStartInfo
            {
                FileName = "lexc",
                Arguments = "content/*.lex --types character,item,quest -t cs -o Assets/Scripts/Generated/",
                UseShellExecute = true
            }
        };
        
        process.Start();
        process.WaitForExit();
        
        if (process.ExitCode != 0)
        {
            throw new BuildFailedException("Lex compilation failed");
        }
    }
}
```

## Troubleshooting

### "Type conflicts"

Each definition type generates a separate class. Use unique type names:
- `tower` → `TowerData`
- `enemy` → `EnemyData`

### "Resources.Load returns null"

Make sure ScriptableObject assets are in a `Resources` folder.

### "Serialization errors"

Lex generates `[System.Serializable]` for nested types. For custom types, add your own serialization.
