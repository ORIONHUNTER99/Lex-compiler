# Godot Integration Guide

This guide shows how to use Lex with Godot 4.x to generate GDScript Resources for your game content.

## Quick Start

### 1. Install Lex

```bash
# Windows
irm https://raw.githubusercontent.com/David-Imperium/Lex-compiler/main/install.ps1 | iex

# Linux/macOS
curl -fsSL https://raw.githubusercontent.com/David-Imperium/Lex-compiler/main/install.sh | bash
```

### 2. Create Your Content

Create a file `content/towers.lex`:

```lex
structure Turret {
    era: Ancient
    name: "Basic Turret"
    cost: { Gold: 50 }
    damage: 10
    range: 2
    attack_speed: 1.0
}

structure Cannon {
    era: Medieval
    name: "Cannon"
    cost: { Gold: 100, Steel: 20 }
    damage: 25
    range: 3
    attack_speed: 0.5
}
```

### 3. Compile to GDScript

```bash
lexc content/towers.lex -t gd -o godot/content_compiled/
```

This generates `godot/content_compiled/towers.gd`:

```gdscript
# Auto-generated by Lex Compiler
class_name TowersData extends Resource

var structures = {}

func _init():
    structures["Turret"] = {
        "id": "Turret",
        "type": "structure",
        "era": "Ancient",
        "name": "Basic Turret",
        "cost": {"Gold": 50},
        "damage": 10,
        "range": 2,
        "attack_speed": 1.0
    }
    structures["Cannon"] = {
        # ...
    }
```

### 4. Load in Godot

Create `autoload/content_loader.gd`:

```gdscript
extends Node

var towers_data: TowersData
var towers: Dictionary

func _ready():
    # Load the generated Resource
    towers_data = load("res://content_compiled/towers.gd").new()
    towers = towers_data.structures

func get_tower(tower_id: String) -> Dictionary:
    return towers.get(tower_id, {})

func get_tower_cost(tower_id: String, resource: String) -> int:
    var tower = get_tower(tower_id)
    return tower.get("cost", {}).get(resource, 0)
```

### 5. Use in Your Game

```gdscript
# scripts/tower.gd
extends Node2D

@export var tower_id: String

var tower_data: Dictionary

func _ready():
    tower_data = ContentLoader.get_tower(tower_id)
    $RangeCircle.shape.radius = tower_data.get("range", 1) * 64
    
func get_damage() -> int:
    return tower_data.get("damage", 0)
```

## Multiple Content Files

For larger projects, organize content in multiple files:

```
content/
├── towers.lex
├── enemies.lex
├── waves.lex
└── upgrades.lex
```

Compile all:

```bash
lexc content/*.lex -t gd -o godot/content_compiled/
```

Each file generates a separate class:
- `towers.lex` → `TowersData`
- `enemies.lex` → `EnemiesData`
- `waves.lex` → `WavesData`

## Using Conditions

Lex supports conditions that can be evaluated at runtime:

```lex
structure SteamFactory {
    era: Steampunk
    cost: { Gold: 100 }
    
    available_if has_technology(SteamEngine) {
    }
}
```

In Godot:

```gdscript
# autoload/game_state.gd
extends Node

var technologies: Array = []
var resources: Dictionary = {}

func has_technology(tech_id: String) -> bool:
    return tech_id in technologies
```

```gdscript
# Check if building is available
func can_build(structure_id: String) -> bool:
    var data = ContentLoader.get_tower(structure_id)
    for condition in data.get("conditions", []):
        if condition.trigger == "available_if":
            return evaluate_condition(condition.expression)
    return true
```

## Hot Reloading (Development)

Use watch mode during development:

```bash
lexc content/*.lex -t gd -o godot/content_compiled/ --watch
```

Changes to `.lex` files will automatically recompile. In Godot, you may need to reload the scene or use `call_deferred("reload")`.

## Best Practices

### 1. Organization

```
godot/
├── autoload/
│   ├── content_loader.gd
│   └── game_state.gd
├── content_compiled/
│   ├── towers.gd
│   ├── enemies.gd
│   └── waves.gd
├── scenes/
└── scripts/

content/
├── towers.lex
├── enemies.lex
└── waves.lex
```

### 2. Type Hints

The generated GDScript uses standard types. You can create typed wrappers:

```gdscript
class_name TowerData extends RefCounted

var id: String
var name: String
var cost: Dictionary
var damage: int
var range: float

static func from_dict(data: Dictionary) -> TowerData:
    var tower = TowerData.new()
    tower.id = data.get("id", "")
    tower.name = data.get("name", "")
    tower.cost = data.get("cost", {})
    tower.damage = data.get("damage", 0)
    tower.range = data.get("range", 1.0)
    return tower
```

### 3. Custom Schema

Define your own types for non-Imperium games:

```bash
lexc content.lex --types tower,enemy,projectile -t gd
```

## Troubleshooting

### "Class name conflicts"

Each `.lex` file generates a class named from the filename. Use unique filenames:
- `td_towers.lex` → `TdTowersData`
- `rpg_items.lex` → `RpgItemsData`

### "Resource not found"

Make sure the output directory is in `res://` and files have `.gd` extension:

```bash
lexc input.lex -t gd -o godot/content_compiled/
```

### "Conditions not evaluating"

Conditions are stored as strings. Use the LexHelper functions or implement your own evaluator:

```gdscript
func evaluate_condition(expr: String) -> bool:
    if expr.begins_with("has_technology("):
        var tech_id = expr.substr(15, expr.length() - 16)
        return GameState.has_technology(tech_id)
    return true
```
